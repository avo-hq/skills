# Avo Skills

A collection of Claude Code skills for Avo developers. These skills help you build, upgrade, and enhance Avo-powered admin panels faster.

## What are Skills?

Skills are prompt templates that teach Claude how to perform specific Avo tasks. They live in `.claude/skills/` and are invoked with slash commands (e.g., `/menu-icons`).

## Available Skills

| Skill | Command | Description |
| ----- | ------- | ----------- |
| Menu Icons   | `/menu-icons`    | Auto-populate Avo menu items with appropriate Tabler icons  |
| Create Skill | `/create-skill`  | Scaffold a new Claude Code skill in this repository         |

## Installation

Copy the `.claude/` directory into your Avo app's root:

```bash
# From the skills repo root
cp -r .claude /path/to/your/rails/app/
```

Or symlink individual skills if you prefer to receive updates via git pull:

```bash
ln -s /path/to/skills/.claude/skills/menu-icons /your/app/.claude/skills/menu-icons
```

## Icon Cache

The `/menu-icons` skill needs a list of available Tabler icons to choose from. It can fetch this live from GitHub at runtime, but it is faster and more reliable to generate a local cache first.

### Generate the cache

```bash
ruby .claude/skills/menu-icons/scripts/fetch_icons.rb
```

This writes `.claude/skills/menu-icons/icons-cache.json` with all 5000+ outline and 1000+ filled Tabler icon names. Run it again any time you want to refresh the list (e.g., after a new Tabler release).

```bash
# Optional: provide a GitHub token to avoid rate limits
ruby .claude/skills/menu-icons/scripts/fetch_icons.rb --token ghp_yourtoken
```

The cache file is checked into version control so your whole team benefits from it without each person needing to run the script.

## Usage

Once installed, run from the Claude Code CLI inside your Avo app:

```
/menu-icons
```

Claude will:
1. Load the icon list from the local cache (or fetch live from GitHub if no cache exists)
2. Read your `config/initializers/avo.rb` menu configuration
3. Add semantically appropriate `icon:` arguments to every item that is missing one
4. Report what was added and why

## Contributing

Each skill lives in its own directory under `.claude/skills/` with a `SKILL.md` prompt file.

```
.claude/
  skills/
    my-skill/
      SKILL.md             ← the skill prompt
      icons-cache.json     ← cached data (generated by script)
      scripts/
        fetch_icons.rb     ← data fetch utilities, co-located with the skill
```
